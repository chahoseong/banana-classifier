# FE-BE 협업 가이드라인

본 문서는 실시간 바나나 숙성도 판별 서비스(Banana Ripe Checker)의 원활한 구현을 위해, 백엔드(FastAPI) 개발자가 프론트엔드(React) 개발자에게 요청하는 기술적 요구사항 및 협의 사항을 정리한 문서입니다. `banana-ripe-checker-prd.md`의 기획 요구사항을 기준으로 작성되었습니다.

---

## 1. 이미지 캡처 및 전송 규격
* **해상도 일치**: 원활한 ML 전처리와 성능 최적화를 위해 Canvas에서 프레임 캡처 시 **반드시 `300x300` 해상도로 고정**하여 추출해 주시기 바랍니다.
* **전송 포맷**: 캡처된 이미지는 바이너리 파일 형태로 담아 `multipart/form-data` 형식으로 API에 전송해 주세요. 파일 용량을 줄이기 위해 `image/jpeg` 포맷(예: quality 0.7~0.8)의 사용을 권장합니다.
* **Key Name**: 폼 데이터 전송 시 파일 파라미터(Key) 이름은 `file` 혹은 명시적으로 협의된 이름을 사용합니다 (예: `formData.append('file', blob);`).

## 2. API 엔드포인트 및 응답 규격
백엔드의 기본 추론 엔드포인트는 다음과 같이 제공됩니다:
* **Endpoint**: `POST /predict`
* **Response Format**: JSON 반환
  
프론트엔드에서는 응답값(JSON)을 수신하여 다음 4가지 핵심 필드를 UI에 매핑해주시면 됩니다.
```json
{
  "is_banana": true,         // 문지기 결과 (바나나 인식 여부)
  "status": "ripe",          // 4단계 숙성도 (unripe, ripe, overripe, dispose)
  "confidence": 0.87,        // 예측 확신도 (0.0 ~ 1.0)
  "message": "바로 먹기 좋은 상태입니다." // 상황별 클라이언트 노출용 메시지 (상세 문구 협의 가능)
}
```

## 3. 데이터 최적화 (조명 강건성)
* 백엔드에서는 BGR 대신 조명 밝기에 강건한 **HSV 색 공간**을 변환하여 이미지의 색상 분포 히스토그램을 추출할 예정입니다.
* 이를 위해, 프론트엔드가 전송하는 해상도(`300x300`)와 화질(크롭 영역 등) 조건이 균일하게 유지되는 것이 매우 중요합니다. 카메라 화상에서 바나나가 가급적 중앙에 오도록 UI 가이드(사각형 가이드라인 등)를 제공해주시면 정확도가 크게 향상됩니다.

## 4. 실시간 전송 로직 제어 (Throttle & 동기식 호출)
가장 중요한 네트워크 부하 방지 및 지연(Latency) 관리 항목입니다.
* **이전 요청 응답 수신 대기**: 무조건적인 고정 주기의 `setInterval` 전송 방식을 **금지**합니다. 프레임을 서버로 전송한 후, **서버의 응답값(200 OK 등)이 돌아온 이후에만 다음 프레임을 캡처하여 전송하는 재귀적(또는 상태 기반) 호출** 방식으로 구현해 주세요.
* **클라이언트 로컬 대기 시간 (Throttle)**: 무한정 프레임 연쇄 전송을 막기 위해, 한 번 결과를 처리한 후 **최소 500ms의 로컬 대기 시간**을 적용한 뒤 다음 프레임을 서버로 보내주시기 바랍니다. (서버의 순수 내부 처리 시간은 300ms 이내를 목표로 합니다).
* **예외/오류 처리 (Error Handling)**: `4xx` 또는 `5xx` 에러가 발생하거나 네트워크가 끊긴 경우, 프론트엔드 화면이 멈추지 않도록 **n초 후 재시도(Retry) 로직**과 함께 사용자에게 직관적인 실패 피드백 안내(토스트 알림 등)를 구현해 주시기 바랍니다.

## 5. UI 상태 분류 및 피드백 매핑
우리가 결정한 숙성도는 총 4가지 단계이며, 프론트엔드 UI(바운딩 박스 색상, 텍스트 상태 등)에 즉각 반영되어야 합니다.
* **`is_banana: false` 일 때**:
  * 회색 또는 무분별한 상태(None)를 나타내는 피드백 (예: "바나나를 카메라 영역에 비춰주세요.")
* **`is_banana: true` 일 때 (`status`에 따른 색상 매핑)**:
  1. `unripe`: **녹색 (Green)** 계열 테두리/텍스트
  2. `ripe`: **노란색 (Yellow)** 계열 테두리/텍스트
  3. `overripe`: **갈색 (Brown)** 계열 테두리/텍스트
  4. `dispose`: **검정색/빨간색 (Black/Red)** 계열 등 경고(버림) 피드백 테두리/텍스트

지연 없는 원활한 서비스 경험을 위해 위 조건들을 기준으로 협업을 부탁드립니다. 추가적인 API 응답 필드나 캔버스 최적화 관련 문의는 언제든 리뷰/답변해 드리겠습니다.
