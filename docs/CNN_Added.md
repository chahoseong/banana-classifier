# CNN 모델 통합 및 모델 선택 기능 가이드

본 문서는 사용자가 UI에서 **CNN(정확도 중시)** 모델과 **SVM(속도 중시)** 모델 중 하나를 선택하여 바나나 숙도를 판별할 수 있도록 프론트엔드와 백엔드를 수정하기 위한 가이드라인입니다.

---

## 1. 모델 개요 (Model Overview)

| 모델명 | 파일명 | 형식 | 특징 | 권장 상황 |
| :--- | :--- | :--- | :--- | :--- |
| **CNN** | `banana_cnn_v1.pth` | PyTorch | 고정밀 (71.5%), 갈색 반점 및 형태 인식 탁월 | 정확한 판별이 필요한 경우 |
| **SVM** | `banana_model_latest.joblib` | Scikit-learn | 고속, 저사양 리소스, 단순 색상 기반 | 빠른 반응성이 필요한 경우 |

---

## 2. 백엔드(Backend) 개발자 전달 사항

### API 엔드포인트 수정 계획
- **Endpoint**: `POST /predict`
- **추가 파라미터**: `model_type: str` (값: `"cnn"` 또는 `"svm"`, 기본값: `"cnn"`)

### 추론 로직 (Inference Process)
1. **모델 로딩**: 
   - `svm` 선택 시: `joblib.load()`를 통해 `.joblib` 파일 로드.
   - `cnn` 선택 시: `torch.load()`를 통해 `.pth` 파일 로드 및 `EfficientNet-B0` 아키텍처 적용.
2. **CNN 전처리**: 
   - CNN 모델 사용 시 이미지를 **(224, 224)** 크기로 리사이징하고, `image_processor.py`의 마스킹 로직을 거친 후 정규화(Normalization) 과정이 필요합니다.
3. **결과 반환**:
   - 응답 필드에 사용된 모델명을 포함하여 프론트엔드에서 표시할 수 있도록 합니다.

---

## 3. 프론트엔드(Frontend) 개발자 전달 사항

### UI 구성 요소 추가
- **모델 선택 컴포넌트**: `st.radio` 또는 `st.selectbox`를 사용하여 사용자가 "정확도 모드 (CNN)"와 "고속 모드 (SVM)"를 선택할 수 있게 합니다.
- **설명 툴팁**: 각 모델의 장단점을 짧게 노출하여 사용자의 선택을 돕습니다.

### API 호출 변경
- 백엔드에 이미지를 전송할 때, 사용자가 선택한 `model_type`을 파라미터에 포함하여 요청합니다.

### 결과 화면 표시
- 판별 결과와 함께 **"사용한 모델: CNN"** 과 같은 정보를 하단에 작게 표시하여 신뢰도를 높입니다.

---

## 4. 라이브러리 의존성 체크
- 백엔드 환경에 `torch` 및 `torchvision`이 설치되어 있어야 합니다. (`requirements.txt` 확인 요망)

---
**주의:** `image_processor.py`는 두 모델 모두에서 공통적으로 배경 제거(Masking)에 활용되므로, 해당 모듈의 경로가 올바르게 참조되었는지 확인하십시오.
