# BE-ML 협업 가이드라인

본 문서는 `banana-ripe-checker-prd.md`의 기획 요구사항을 바탕으로 백엔드(FastAPI) 개발자와 머신러닝(ML) 개발자가 원활한 협업을 위해 사전에 논의되어야 했던 내용들을 **협의 완료(결정)된 상태**로 정리한 기술 가이드라인입니다.

## 1. 전처리(Preprocessing) 및 특징 추출(Feature Extraction) 책임 소관
* **Feature Extraction 주체 (결정)**: ML 파트가 바나나 영역 마스킹 및 HSV 변환 등 전처리를 수행하는 `ml/src/preprocessing/image_processor.py`를 제공하면, **백엔드(FastAPI)가 해당 파이썬 모듈을 API 라우터 내에서 import하여 예측 파이프라인의 일부로 실행**합니다.
* **`is_banana` (바나나 여부 판단) 로직 (결정)**: 바나나가 아닌 배경(None)을 걸러내는 **문지기 역할(`is_banana`) 로직은 전적으로 백엔드(BE)에서 구현 및 전담**합니다. 따라서 ML 모델은 순수하게 4클래스 분류에만 집중하여 학습/평가 루프를 돌릴 수 있습니다.
* **입력 규격 (결정)**: 프론트엔드가 송신한 이미지는 백엔드에서 **`300x300`의 Numpy Array(BGR 포맷)**로 변환된 후 `image_processor`에 넘겨집니다.

## 2. 추론 (Inference) 인터페이스 및 모델 연동
* **모델 아티팩트 배포 및 로드 방식 (결정)**: ML 파트에서 학습 완료된 분류 모델(`.joblib` 또는 `.pkl` 사용 권장)을 **`ml/artifacts/banana_model_latest.joblib`** 지정된 단일 경로로 저장합니다. 백엔드 서버는 시작(`lifespan` 혹은 `startup` 단계) 시 이 경로에서 모델을 1회만 메모리에 캐싱(Load)합니다.
* **API 호출 구조 및 출력값 (결정)**: 
  * 백엔드의 FastAPI 코드는 ML 파트가 제공한 추론 함수를 호출하여, **예측된 상태 클래스(Status 문자열)와 확신도(Confidence 점수 0.0~1.0 튜플)**를 리턴받습니다.
  * 최종적으로 백엔드는 `is_banana` 필드와 함께 JSON을 프론트엔드로 응답합니다.
* **클래스 상태명 규칙 (결정)**: FE 연동과 혼선을 피하기 위해 모델이 뱉는 상태 문자열은 **모두 소문자 영문으로 통일**합니다.
  * 스펙: `unripe`, `ripe`, `overripe`, `dispose`

## 3. 지연시간(Latency) 예산 관리 (최대 500ms)
사용자 체감 성능 향상을 위해 총 응답 시간을 제한(500ms)하므로, 각 파트에 할당된 처리 시간 목표를 명시합니다. 
* **네트워크 대기:** FE에서 최소 500ms Throttle 로컬 딜레이 지정 (서버 딜레이 제외)
* **순수 서버 처리 시간 목표 (결정)**: 서버의 총 처리 시간은 **300ms 이내**로 완료되어야 합니다.
  * **BE 담당**: FastAPI 이미지 수신 및 OpenCV BGR 파싱 -> **50ms 이내**
  * **ML 담당**: ROI 추출 + 특징 변환 + K-NN/SVM 추론 시간 전체 -> **250ms 이내**

## 4. 모델 재학습을 위한 데이터 로깅 (결정 사항)
ML 성능 고도화를 위해 추론된 데이터의 로깅 파이프라인(저장 위치 및 방법)을 다음과 같이 정합니다.
* **로깅 조건 및 경로**: 백엔드는 추론 결과 `confidence`가 기준치(최초 **0.6 미만**)인 불안정한 추론 이미지에 한해, 비동기 백그라운드 태스크를 통해 다음 경로에 이미지를 원본(300x300) 그대로 저장합니다.
  * 경로 스키마 예: `dataset/raw/low_confidence/{timestamp}_pred_{status}.jpg`
* **ML 재학습 연동**: ML 담당자는 위 폴더에 적재된 데이터를 주기적으로 수거하여 오답 노트를 라벨링하고 Retraining(재학습) 파이프라인에 투입합니다.

## 5. 의존성 (Dependencies) 및 패키지 주도권
* **공통 오픈소스 버전 기준 (결정)**: 백엔드가 ML 추론 파이프라인을 직접 실행하므로, ML 예측을 보장하는 라이브러리(`scikit-learn`, `opencv-python-headless`, `numpy` 등)의 코어 버전은 **ML 개발자가 결정 및 확정**합니다.
* BE 개발자는 `ml/requirements.txt`에 명시된 주요 데이터 패키지들의 버전을 최우선으로 존중하여, `backend/requirements.txt` 또는 Dockerfile/서버 환경에 통합/설치합니다. (의존성 충돌로 인한 500 에러를 원천 차단하기 위함입니다)
