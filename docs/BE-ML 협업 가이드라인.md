# Backend - ML 개발자 협의 사항

본 문서는 `banana-ripe-checker-prd.md`의 기획 요구사항을 바탕으로 백엔드(FastAPI) 개발자와 머신러닝(ML) 개발자가 원활한 협업을 위해 사전에 논의하고 합의해야 하는 핵심 내용들을 정리한 것입니다.

## 1. 이미지 처리 및 특징 추출 (Feature Extraction) 책임 소관
* **Feature Extraction 주체**: 클라이언트로부터 업로드된 이미지를 OpenCV 형식으로 변환하고 HSV 색 공간에서 주요 특징을 추출하는 전처리 로직은 **백엔드(FastAPI)**가 담당할지, **ML 모듈** 내부에서 전담할지 경계를 명확히 해야 합니다. (권장: ML이 `image_processor.py`를 제공하고, BE가 이를 import하여 파이프라인 구성)
* **`is_banana` (배경/바나나 이진 분류) 로직**: 바나나가 아닌 배경(None)을 걸러내는 **문지기 역할(`is_banana`) 로직은 전적으로 백엔드(BE)에서 구현 및 전담**합니다. 따라서 ML 모델은 4클래스(`Unripe`, `Ripe`, `Overripe`, `Dispose`) 분류에만 집중하여 학습을 진행하면 됩니다.
* **입력 규격 합의**: 모든 이미지 연산은 `300x300` 해상도를 기준으로 진행됩니다. 전처리 시 리사이징과 캔버스 비율 등에 대한 가정을 양측이 동일하게 공유해야 합니다.

## 2. 추론 (Inference) 인터페이스 및 모델 연동
* **모델 로드 방식**: ML 파트에서 학습을 마친 모델 파라미터 파일(예: `.pkl`, `.joblib`)을 `ml/artifacts/` 디렉터리에 배포할 때, 백엔드 서버(FastAPI)가 시작 시 이를 메모리에 어떻게 로드할지 경로를 협의해야 합니다.
* **API 입출력 규격**: 
  * 모델 추론 함수의 **입력 데이터 구조** (예: 단일 배열, 사전(Dictionary), Numpy Array 등)
  * 모델 추론 함수의 **반환 결과 포맷** (모델이 4단계 상태와 확신도를 반환하면 백엔드에서 `is_banana` 필드를 추가하여 최종 JSON 조합)
  * **출력 예시**: `{"status": "Ripe", "confidence": 0.87}` (ML 모듈의 반환값 기준)
* **클래스 상태명 규칙**: 서비스와 모델이 동일하게 인식할 수 있도록 문자열의 대소문자 규약을 통일해야 합니다. (예: `Unripe`, `Ripe`, `Overripe`, `Dispose`)

## 3. 목표 지연시간 (Latency) 확보 방안 (최대 500ms)
* **목표 성능**: 요구사항 상 이미지 업로드부터 결과 반환까지의 **총 응답 시간은 최대 500ms**로 제한됩니다.
* **병목 관리**: FastAPI 이미지 파싱 속도, OpenCV 이미지 변환 및 색상 특징 계산 속도, K-NN / SVM 추론 속도를 합산하여 500ms 예산 안에 안정적으로 들어오는지 양 파트가 모니터링하여 병목을 최적화해야 합니다.

## 4. 의존성 (Dependencies) 및 실행 환경 충돌 방지
* **공통 라이브러리 일치**: 백엔드 API가 ML 추론 모듈을 임포트해서 실행하려면 공통된 버전을 보장해야 합니다. `scikit-learn`, `opencv-python`, `numpy` 등 핵심 모듈을 중심으로 `backend/requirements.txt`와 `ml/requirements.txt` 간 버전 충돌이 발생하지 않도록 패키지 관리에 유의합니다.

## 5. 데이터 로깅 및 재학습 파이프라인 (향후 고도화)
* **로깅 위치 및 포맷**: 추론 결과의 확신도(Confidence)가 낮거나 모델이 예측한 값과 실제 값이 불일치할 가능성이 있는 이미지를 백엔드가 어떻게 별도로 서버에 보관/기록할지 정해야 합니다 (폴더명 예: `dataset/raw/low_confidence/`).
* **재학습(Retraining) 연동**: 저장된 이미지를 향후 ML 개발자가 어떻게 손쉽게 가져와 라벨링하고 다시 모델 학습 루프에 공급할 것인지 데이터 파이프라인의 연계 방안을 설계해야 합니다.
