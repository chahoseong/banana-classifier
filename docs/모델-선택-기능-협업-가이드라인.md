# 바나나 숙성도 분류 모델 선택 기능 협업 가이드라인

본 문서는 사용자가 바나나 숙성도 분석 시 사용할 모델을 직접 선택할 수 있도록 기능을 고도화하기 위해, 프론트엔드와 백엔드 간의 기술적 협업 규격을 정의합니다.

---

## 1. 모델 리스트 조회 API

프론트엔드에서 사용자에게 선택 가능한 모델 목록을 보여주기 위해 사용됩니다.

- **Endpoint**: `GET /models`
- **Response Format**: JSON List
- **Response Example**:
```json
[
  {
    "id": "baseline",
    "name": "Baseline Model (KNN)",
    "description": "이미지 색상 분포를 기반으로 한 기본 분석 모델",
    "type": "classic_ml"
  },
  {
    "id": "cnn_mobilenet",
    "name": "MobileNetV2 (CNN)",
    "description": "딥러닝 알고리즘을 사용한 고정밀 분석 모델",
    "type": "deep_learning"
  }
]
```

## 2. 숙성도 추론 API 확장 (모델 선택 지원)

기존 추론 API 요청 시 어떤 모델을 사용할지 명시하는 파라미터가 추가되어야 합니다.

- **Endpoint**: `POST /predict`
- **Request Format**: `multipart/form-data`
- **Parameters**:
  - `file`: 이미지 파일 (300x300 권장)
  - `model_id` (Optional/Default: `baseline`): 사용할 모델의 고유 ID (위 리스트 API의 `id` 값)
- **Response Format**: 기존 규격 유지 (예: `is_banana`, `status`, `confidence`, `message`)

## 3. 백엔드 모델 관리 규칙

- **동적 로딩/수위칭**: 요청받은 `model_id`에 해당하는 모델 인스턴스(singleton)를 사용하여 추론을 수행해야 합니다.
- **예외 처리**: 존재하지 않는 `model_id`가 요청될 경우 `400 Bad Request`와 함께 적절한 에러 메시지를 반환해 주세요.

## 4. 프론트엔드 구현 사항

- 카메라 화면 상단에 백엔드에서 내려준 `/models` 리스트를 기반으로 한 드롭다운 UI 제공
- 사용자가 선택한 모델 ID를 로컬 상태로 저장하고, `/predict` 요청 시마다 해당 ID를 파라미터로 포함하여 전송
- 모델 전환 시 사용자 경험을 위해 로딩 상태 표시 등 최적화
